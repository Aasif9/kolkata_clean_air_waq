## 2. Routing Algorithm

### Algorithm: Dijkstra's Shortest Path

**Implementation**: NetworkX library (`nx.shortest_path`)
**File**: `simple_router.py`

### Two Route Types Calculated

#### 1. Fastest Route
- **Algorithm**: Dijkstra with `travel_time` weight
- **Objective**: Minimize travel time
- **Weight Function**: `weight = travel_time`

#### 2. Cleanest Route
- **Algorithm**: Dijkstra with custom pollution weight
- **Objective**: Minimize pollution exposure
- **Weight Function**:
```python
weight = travel_time * (1 + (pollution_factor * (aqi / 100)))
```

Where:
- `travel_time`: Edge travel time in minutes
- `pollution_factor`: User-adjustable sensitivity (default 2.0)
- `aqi`: Air Quality Index at edge midpoint (interpolated from stations)

### AQI Interpolation

**Method**: Inverse Distance Weighting (IDW)
**Implementation**: `dummy_aqi_interpolator.py`
**Formula**:
```python
weight = 1 / (distance + 0.1)
interpolated_aqi = sum(weight * aqi_value) / sum(weights)
```

**Purpose**: Estimate AQI at any point by averaging nearby station values weighted by distance

## 3. Performance Analysis: Why 2 Minutes?

### Root Cause: Inefficient Nearest Node Search

**Location**: `simple_router.py`, line 78-96, method `_find_graph_node()`

**Problem Code:**
```python
def _find_graph_node(self, lat, lon):
    """Find nearest graph node to coordinates"""
    min_dist = float('inf')
    nearest_node = None
    
    for node in self.graph.nodes():  # ITERATES THROUGH ALL 31,894 NODES
        node_data = self.graph.nodes[node]
        node_lat = node_data['y']
        node_lon = node_data['x']
        
        distance = geodesic((lat, lon), (node_lat, node_lon)).meters
        
        if distance < min_dist:
            min_dist = distance
            nearest_node = node
    
    return nearest_node
```

### Performance Breakdown

**Graph Size**: 31,894 nodes
**Operations per Route Calculation**:
1. Find start node: 31,894 distance calculations
2. Find end node: 31,894 distance calculations
3. Fastest route: Dijkstra on 31,894 nodes
4. Cleanest route: Dijkstra with AQI interpolation at each edge
5. Path analysis: AQI interpolation for each edge (84-105 edges)

**Total per request**: ~64,000+ geodesic distance calculations

**Geodesic calculation**: Uses `geopy.distance.geodesic()` which is computationally expensive (Haversine formula)

### Why It Takes 2 Minutes

1. **Linear Search**: O(n) complexity for nearest node (n = 31,894)
2. **Expensive Distance Calc**: Geodesic distance is slower than Euclidean
3. **Multiple Calls**: Called 2+ times per route calculation
4. **Dijkstra Overhead**: NetworkX Dijkstra on large graph
5. **AQI Interpolation**: IDW calculation at every edge midpoint

### Estimated Time Distribution
- Nearest node search (2x): ~60 seconds
- Dijkstra pathfinding (2x): ~40 seconds
- Path analysis with AQI: ~20 seconds
- **Total**: ~120 seconds (2 minutes)

## 5. Route Calculation Flow

### Step-by-Step Process

```
User Request (coordinates)
    ↓
1. Find Nearest Graph Nodes (SLOW - 31,894 iterations each)
    ↓
2. Calculate Fastest Route (Dijkstra on travel_time)
    ↓
3. Calculate Cleanest Route (Dijkstra with pollution weight)
    ↓
4. Analyze Both Routes
    - Calculate distance (geodesic per edge)
    - Interpolate AQI at edge midpoints (IDW)
    - Compute statistics
    ↓
5. Return JSON Response
```

### Data Flow

```
kolkata_road_network.pkl (31,894 nodes)
    ↓
SimplePollutionRouter
    ↓
DummyAQIInterpolator
    ↓
kolkata_dummy_stations.json (25 stations)
    ↓
IDW Interpolation → AQI values at any point
    ↓
Dijkstra Algorithm → Optimal paths
    ↓
Response with coordinates & analysis
```

## 6. Performance Optimization Recommendations

### Critical Fix: Spatial Indexing

**Current**: O(n) linear search through all nodes
**Recommended**: O(log n) spatial indexing with R-tree or KD-tree

**Implementation**:
```python
from scipy.spatial import cKDTree

# Build index once
node_coords = np.array([[node['y'], node['x']] for node in graph.nodes()])
tree = cKDTree(node_coords)

# Fast nearest node search
def _find_graph_node_fast(self, lat, lon):
    dist, idx = tree.query([lat, lon])
    return list(graph.nodes())[idx]
```

**Expected Improvement**: 60 seconds → 0.1 seconds (600x faster)

### Secondary Optimizations

1. **Precompute edge midpoints**: Store in graph to avoid repeated calculation
2. **Cache AQI interpolations**: Memoize AQI values for coordinates
3. **Use Euclidean distance**: Faster than geodesic for node search (acceptable for small distances)
4. **Parallel path calculation**: Calculate both routes simultaneously
5. **Reduce graph size**: Filter to relevant region before routing
